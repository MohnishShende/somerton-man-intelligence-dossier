# Security Policy

This repository does not process private production data. Security issues are still possible, especially around malicious files, build scripts, or external links.

Do not submit:

- Private personal data about living people unless it is already lawfully public, relevant, and ethically necessary
- Credentials, API keys, or private archive access tokens
- Unredacted sensitive correspondence
- Malware, tracking links, or unsafe executable files

Report security concerns by opening a private advisory when the repository is public, or by contacting the maintainer directly.
