# Versioning and Release Strategy

## Repository Versions

Use semantic versioning for infrastructure:

- Patch: corrections, metadata fixes, minor build changes
- Minor: new verified source batches, new chapter drafts, expanded registers
- Major: public dossier release or major methodological revision

## Dossier Releases

Each public release should include:

- Compiled PDF
- Source catalogue snapshot
- Claim register snapshot
- Changelog
- Known limitations

## Suggested Tags

- `v0.1.0-foundation`
- `v0.2.0-source-intake`
- `v0.3.0-verification-batch-1`
- `v1.0.0-public-assessment`

## Long-Term Maintenance

- Review broken links quarterly.
- Re-run builds after TeX package updates.
- Archive cited web sources.
- Keep correction issues open until resolved in the claim register and manuscript.
